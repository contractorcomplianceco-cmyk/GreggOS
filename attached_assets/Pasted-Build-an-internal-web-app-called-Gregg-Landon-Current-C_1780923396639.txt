Build an internal web app called “Gregg + Landon Current Client Cockpit.”

Purpose:
This app is an internal Replit-first cockpit for Contractor Compliance Authority. It helps Gregg manage high-level current-client relationships and helps Landon process Gregg’s RingCentral call notes into organized summaries, tasks, missing information, opportunity signals, and escalation flags.

This is the first prototype. Do not connect to live Zoho, RingCentral, WorkDrive, or Company Brain yet. Store all data locally in browser state or localStorage. Include realistic sample data.

Tech:
Use React with Vite.
Use clean, modern, operational UI.
The app should feel like a working internal dashboard, not a marketing page.
Use dense, organized layouts suitable for daily work.
Use tabs or sidebar navigation.
Use copy buttons for generated text.
Use export JSON buttons where useful.

Primary users:
1. Gregg Taylor
   Role: President / Chief Contractor Relations & Placement Officer.
   Needs a simple executive view of current clients, urgent follow-ups, at-risk accounts, qualifier/placement mentions, and items needing his judgment.

2. Landon Taylor
   Role: Executive Support & Placement Operations Coordinator.
   Needs a work queue for RingCentral notes, CRM-ready summaries, follow-up tasks, missing information, and escalation routing.

Pilot scope:
Current clients only for the first version.
Qualifier, placement, audit, monitoring, renewal, expansion, and client-save opportunities should be captured only when they surface inside current-client conversations.

Do not include:
- Final pricing
- Legal advice
- Final compliance opinions
- Refund approvals
- Placement approvals
- Live CRM writes
- Live RingCentral API calls
- Live WorkDrive access
- Company Brain auto-updates

Required behavior:

When Landon processes a call note:
1. He selects the client.
2. He pastes the raw RingCentral note.
3. He fills or edits summary fields.
4. The app generates:
   - CRM-ready note
   - client-facing follow-up draft
   - task list
   - JSON export
5. He can mark the note as:
   - Gregg review
   - CRM-ready
   - Copied to CRM
   - Escalated
   - Archived
6. The related client profile updates.
7. Gregg Today updates if Gregg action is needed.

CRM-ready note format:
Use a factual paragraph format:
“Call with [contact/client] on [date]. Discussed [summary]. Client concern/request: [concern]. Missing information: [missing info]. Next steps: [next actions]. Owner: [next owner]. Due date: [due date]. Opportunity signals: [signals]. Escalation: [escalation status].”

Client follow-up draft format:
“Hi [contact name], thank you for speaking with Gregg today. Based on the conversation, our next step is [next action]. We are currently waiting on [missing information if any]. We will follow up by [due date if available].”

Task list format:
Create separate task items from next actions. Each task should have:
- title
- owner
- due date
- priority
- source call note

Decision boundaries:
Add visible internal reminder language somewhere in the app:
“This cockpit organizes relationship follow-through. It does not approve pricing, refunds, legal advice, compliance opinions, qualifier placements, or final client commitments.”

Also include:
“RingCentral AI notes are draft source material until reviewed.”

Acceptance criteria:
- Gregg can open one screen and see which current clients need attention today.
- Landon can process a RingCentral note into a clean CRM-ready summary in under five minutes.
- Every meaningful call can produce at least one of: no action needed, follow-up task, missing information request, escalation, or opportunity signal.
- The cockpit clearly separates draft RingCentral notes from reviewed summaries.
- Weekly Review shows overdue items, stalled clients, high-risk accounts, and opportunities.
- Copy buttons work for CRM note, client follow-up draft, task list, and JSON.
- The prototype does not write to live Zoho, RingCentral, WorkDrive, or Company Brain.
- Role boundaries are visible and clear.
