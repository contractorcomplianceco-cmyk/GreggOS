Build an internal executive operating system called:

“GreggOS Command Center”

Purpose:
This is NOT a CRM or assistant tool.

This is an executive-level command and oversight system for Gregg Taylor’s leadership role inside Contractor Compliance Authority.

Gregg is NOT a single point of contact anymore.
He is responsible for:
- Company-wide client satisfaction oversight
- Early detection of client risk and anomalies
- Relationship growth strategy and intervention
- Executive escalation handling
- Opportunity identification across all clients
- High-level operational awareness

This system must feel like a CEO command dashboard.

Tech Stack:
- React (Vite)
- Local-first data (localStorage)
- Clean, high-density executive UI
- Sidebar + modular dashboards
- Fast filtering, search, and alerting
- No backend required for MVP

CORE DESIGN PRINCIPLE:
“Show Gregg what is going wrong, what is about to go wrong, and what can be expanded.”

--------------------------------------------
MODULE 1: EXECUTIVE COMMAND DASHBOARD (HOME)
--------------------------------------------

This is the main screen.

Show:

1. RED FLAGS (Critical Alerts)
- At-risk clients
- Negative sentiment signals
- Missed follow-ups
- Escalations
- Compliance risk flags
- Stalled high-value relationships
- Sudden drop in engagement
- Unresolved complaints

2. EARLY ANOMALY DETECTION
- Clients with unusual silence periods
- Repeated support issues
- Increased documentation delays
- Pricing sensitivity signals
- Qualifier instability indicators
- Repeat follow-ups without resolution

3. CLIENT SATISFACTION HEALTH MAP
- Green / Yellow / Red status per client
- Trend indicator (improving / stable / declining)
- Last interaction timestamp
- Sentiment summary (from notes)

4. OPPORTUNITY RADAR
- Upsell opportunities
- Renewal expansion signals
- Relationship deepening opportunities
- Event/travel-worthy clients
- Strategic accounts

5. TODAY’S EXECUTIVE PRIORITIES
- Only what Gregg must personally be aware of
- NOT operational tasks
- NOT low-level tasks
